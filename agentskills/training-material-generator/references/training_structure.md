## 概览
本文档提供培训材料的标准结构定义和内容格式规范，用于指导生成高质量的培训文档。

## 标准结构

### 基础结构（适用于2-4小时的培训）
```
封面
├── 培训标题
├── 副标题/培训目标
└── 制作人信息

第一章：课程导论
├── 培训目标说明
├── 课程概述
└── 学习方法建议

第二章：核心内容1
├── 概念讲解
├── 方法介绍
└── 实例说明

第三章：核心内容2
├── 理论基础
├── 实践应用
└── 常见问题

第四章：总结与实践
├── 知识回顾
├── 实践建议
└── 后续学习资源
```

### 扩展结构（适用于半天或一天的培训）
在基础结构基础上，根据需要增加：
- 实操演练章节
- 案例分析章节
- 互动讨论章节
- 评估测试章节

```json
          "content": "本课程分为理论学习和实践操作两个部分。理论学习部分涵盖数据分析的基本原理和方法，实践操作部分通过实际案例演示如何使用Python工具进行数据分析。"

```

### 字段详细说明

#### 顶层字段
| 字段名 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| title | string | 是 | 培训材料的主标题 |
| subtitle | string | 否 | 副标题或培训目标描述 |
| author | string | 否 | 制作人或讲师信息 |
| chapters | array | 是 | 章节数组，至少包含一个章节 |

#### 章节对象（chapter）
| 字段名 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| chapter_title | string | 是 | 章节标题 |
| sections | array | 是 | 小节数组 |

#### 小节对象（section）
| 字段名 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| section_title | string | 是 | 小节标题 |
| content | string | 否 | 详细内容，支持多段落（用\n\n分隔） |
| points | array | 否 | 关键要点列表 |

## 内容设计原则

### 1. 结构清晰
- 每个章节有明确的主题和目标
- 小节之间逻辑递进
- 内容层次分明，易于导航

### 2. 内容适量
- 单个章节控制在2000-3000字
- 单个小节控制在500-1000字
- 避免信息过载

### 3. 实用导向
- 理论与实践结合
- 提供具体案例和示例
- 强调实际应用场景

### 4. 适应听众
- 根据听众背景调整专业术语使用
- 初学者：多用实例，少用抽象概念
- 有基础者：增加深度，提供进阶内容

### 5. 可操作性
- 提供明确的操作步骤
- 列出关键要点和注意事项
- 给出实践建议

## 验证规则

### JSON格式验证
1. 确保所有引号成对出现
2. 确保所有括号正确闭合
3. 字符串中的特殊字符需要转义（如换行符用\n）
4. 最后一个元素后不能有逗号

### 内容完整性验证
1. 必须包含title和chapters字段
2. 每个章节必须有chapter_title
3. 每个小节必须有section_title
4. content和points至少有一个存在

### 内容质量验证
1. 章节标题具有概括性
2. 小节内容逻辑连贯
3. 要点列表简明扼要
4. 段落长度适中（一般不超过200字）

## 常见问题

### Q: 如何处理长文档？
A: 将内容拆分为多个章节，每个章节聚焦一个主题。必要时使用子小节进一步细分。

### Q: 如何添加代码示例？
A: 在content字段中使用专门的段落描述代码，或创建单独的代码示例章节。

### Q: 如何处理多语言内容？
A: 在同一文档中使用统一的中文或英文，避免中英文混用影响格式。

### Q: 如何生成目录？
A: 当前版本脚本不自动生成目录，如需目录可手动在封面后添加目录页。
