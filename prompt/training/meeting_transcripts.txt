刘*：我们今天这个会，主要还是围绕一个问题，就是今年要不要搞一场智能体相关的大赛活动。如果要搞，是偏内部、偏生态，还是对外公开的。现在行业里 Agent 这个概念已经很热了，我们不能完全没有声音。
周*：对，其实从市场这边看，去年下半年开始，各家都在做大模型应用大赛，但真正把“多智能体”“Agent 编排”作为核心卖点的还不算多。如果我们现在切入，时间点还可以。
赵*：我先泼点冷水啊。现在平台这块，其实还在快速迭代阶段，有些能力我们自己内部用没问题，但一旦对外，文档、稳定性、兼容性压力会非常大。
孙*：但从开发者生态的角度，如果一直等平台“完全稳定”，那可能永远也等不到。很多生态其实就是在不完美中慢慢跑起来的。
刘*：所以这里面其实有一个取舍，是我们把比赛当成一次“展示成熟能力”，还是当成一次“共创+验证”。
陈**：我比较关心的是，如果定位是共创，那外部预期要管理好，不然开发者会觉得体验不好，反而影响品牌。
周*：对，所以我刚才也在想，是不是第一届不要把调子定得太高，比如就叫“智能体创新挑战赛”，而不是“全国大赛”那种。
刘*：名字和定位我们后面可以再讨论。现在先看形式。是线上为主，还是线上线下结合？
陈**：如果线下的话，成本和人力都会上来，而且现在时间也比较紧。
赵*：从研发支持角度，我强烈建议第一届纯线上。线下答辩、Demo Day 对技术支持压力太大。
孙*：线上的话，其实也可以设计一些直播分享、技术讲堂，增强参与感，不一定非要线下。
刘*：那目前大家比较倾向的是：第一届，线上为主，规模可控，对吧？
周*：但线上也分很多种。是开放报名，还是定向邀请？如果完全开放，参赛质量可能参差不齐。
孙*：我建议可以结合，比如公开报名，但设置一个基础门槛，比如必须使用我们的 Agent 框架，或者完成一个基础任务。
赵*：这里我有个技术上的担忧。如果强制用我们的框架，那我们至少要保证 API 在比赛周期内不大改，不然选手会很痛苦。
刘*：那比赛周期大概多久比较合适？
陈**：从运营经验看，至少 4 到 6 周，不然大家来不及做东西。
赵*：6 周的话，对研发来说就意味着 6 周基本 freeze 接口，这个需要内部评估。
黄*：我补充一句。我刚进来听了一下，如果是比赛场景，其实可以考虑提供一个“比赛专用环境”，接口版本固定，不影响主线迭代。
赵*：这个方案可以考虑，但需要额外环境和运维资源。
刘*：这个点先记下来，算是一个可选方案。
周*：还有一个是赛道设计。刚才刘*提到分赛道，我觉得这个挺重要的。不然纯技术比拼，业务方可能参与感不强。
孙*：我也赞同。比如一个“业务应用赛道”，一个“技术创新赛道”，评审标准可以不一样。
陈**：但我还是要提醒一下，赛道一多，规则、评审、奖项设计都会变复杂。我们人手其实有限。
刘*：那有没有可能先做两个赛道，不超过这个数量？
周*：我觉得两个是可以接受的，再多就容易失控。
赵*：技术上两个赛道没问题，但评审要提前想清楚，不然最后争议会很多。
孙*：评审这块，可以考虑内部专家 + 外部嘉宾结合，提高公信力。
刘*：预算呢？现在有没有一个大概判断？
周*：预算目前还没有正式批，但如果是轻量线上赛，我估计可以控制在一个相对合理的范围。奖金不一定要特别高，关键是技术影响力。
陈**：但不管怎样，至少需要一个明确的预算区间，不然很多事情没法启动。
刘*：这个我来协调。
周*：还有时间点。如果目标是上半年，那宣传最晚三月要启动，反推的话，方案和规则可能两周内要有一个初版。
赵*：两周内给“原则性方案”可以，但细节肯定还要后面补。
孙*：我这边可以先做一些开发者非正式调研，看看大家对 Agent 大赛的兴趣点。
刘*：好，那我们是不是可以先形成一个阶段性共识：
第一，今年上半年做；
第二，第一届以线上为主、轻量化；
第三，定位偏创新和生态，不追求一步到位。
周*：我同意。
赵*：在技术风险可控前提下，我也可以接受。
陈**：那我希望下周能尽快把赛制轮廓定下来，不然资源评估不好做。
刘*：OK，那这样。下周我拉一个小会，专门细化赛道、规则和预算。今天先到这里，大家会后把关键点整理给会议助理。
